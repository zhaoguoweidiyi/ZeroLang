# 零点语言（0・）使用文档

## 1. 语言概述

零点语言是一种纯数字指令语言，专为跨平台开发设计。它使用4位数字指令格式，不包含任何人类语言、字母或符号，所有操作均通过数字指令实现。

### 1.1 设计目标
- 跨平台兼容性：支持Windows、Linux、Android和微控制器
- 纯数字指令：仅使用0-9数字，无其他字符
- 简洁高效：4位数字指令格式，易于解析和执行
- 图灵完备：能够实现任何可计算的算法

### 1.2 应用场景
- 嵌入式系统开发
- 跨平台应用开发
- 教育和学习编程基础
- AI代码生成和执行

## 2. 语法规则

### 2.1 指令格式
零点语言使用4位数字指令，格式为：`op a b dst`
- `op`：操作码（1-26）
- `a`：第一个操作数
- `b`：第二个操作数
- `dst`：目标操作数

### 2.2 操作数类型
- 0-9：通用寄存器
- 10+：内存地址或立即数（具体取决于指令）

### 2.3 程序结构
- 程序由一系列4位数字指令组成
- 指令之间无分隔符，连续书写
- 程序从第一条指令开始执行
- 指令按顺序执行，直到遇到退出指令

## 3. 指令集详解

### 3.1 数据传送指令

| 指令码 | 指令名称 | 功能 | 示例 |
|-------|---------|------|------|
| 1 | 立即数赋值 | 将立即数赋值给寄存器 | 1005 → 将5赋值给寄存器0 |
| 2 | 寄存器复制 | 将一个寄存器的值复制到另一个寄存器 | 2012 → 将寄存器0的值复制到寄存器2 |
| 19 | 写内存/硬件 | 将寄存器值写入内存或硬件 | 19012 → 将寄存器0的值写入地址12 |
| 20 | 读内存/硬件 | 从内存或硬件读取值到寄存器 | 20120 → 从地址12读取值到寄存器0 |

### 3.2 算术运算指令

| 指令码 | 指令名称 | 功能 | 示例 |
|-------|---------|------|------|
| 3 | 加法 | 两个寄存器值相加，结果存入目标寄存器 | 3012 → 寄存器0 + 寄存器1 → 寄存器2 |
| 4 | 减法 | 两个寄存器值相减，结果存入目标寄存器 | 4012 → 寄存器0 - 寄存器1 → 寄存器2 |
| 5 | 乘法 | 两个寄存器值相乘，结果存入目标寄存器 | 5012 → 寄存器0 × 寄存器1 → 寄存器2 |
| 6 | 除法 | 两个寄存器值相除，结果存入目标寄存器 | 6012 → 寄存器0 ÷ 寄存器1 → 寄存器2 |
| 7 | 取模 | 两个寄存器值取模，结果存入目标寄存器 | 7012 → 寄存器0 % 寄存器1 → 寄存器2 |

### 3.3 比较运算指令

|-------|---------|------|------|

| 10 | 大于 | 比较第一个寄存器值是否大于第二个，结果存入目标寄存器 | 10012 → 寄存器0 > 寄存器1 → 寄存器2 |
| 11 | 小于 | 比较第一个寄存器值是否小于第二个，结果存入目标寄存器 | 11012 → 寄存器0 < 寄存器1 → 寄存器2 |
| 12 | 大于等于 | 比较第一个寄存器值是否大于等于第二个，结果存入目标寄存器 | 12012 → 寄存器0 >= 寄存器1 → 寄存器2 |

## 5. 编译器使用指南

### 5.1 编译器安装
1. 确保已安装Go语言环境
2. 克隆零点语言编译器代码
3. 编译编译器：`go build -o zero-compiler.exe compiler/main.go compiler/compiler.go`

### 5.2 编译命令

```bash
zero-compiler.exe <input_file> <output_file> <platform>
```

参数说明：
- `<input_file>`：零点语言源代码文件
- `<output_file>`：编译后的可执行文件
- `<platform>`：目标平台（windows, linux, arm64, cortex-m）

### 5.3 示例编译命令

```bash
# 编译Windows x86_64可执行文件
zero-compiler.exe program.txt output.exe windows

# 编译Linux x86_64可执行文件
zero-compiler.exe program.txt output linux

# 编译ARM64可执行文件
zero-compiler.exe program.txt output arm64

# 编译Cortex-M固件
zero-compiler.exe program.txt output.bin cortex-m
```

## 8. 常见问题

### 8.1 指令格式错误
- 确保每条指令都是4位数字
- 检查操作码是否在1-26范围内
- 确保操作数使用正确的寄存器编号

### 8.2 编译失败
- 检查源代码是否符合语法规则
- 确保目标平台支持所需功能
- 检查内存访问是否合法

### 8.3 运行时错误
- 检查除数是否为0
- 确保内存访问地址有效
- 检查函数调用参数是否正确
