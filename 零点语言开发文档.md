# 零点语言（0・）开发文档

## 1. 语言概述

零点语言是一种极简的数字指令语言，专为全平台编译器设计，支持生成各平台原生机器码。该语言仅使用数字指令，不包含任何人类语言、字母、符号等内容，所有操作均通过4位数字指令实现。

## 2. 语言规则

### 2.1 指令格式

每条指令由4个数字组成，格式为：
```
指令 参数1 参数2 目标
```

其中：
- 指令：0-15之间的数字，代表不同的操作
- 参数1：操作的第一个参数
- 参数2：操作的第二个参数
- 目标：操作结果的存储位置（通常是寄存器）

### 2.2 指令集

| 指令码 | 指令名称 | 功能描述 | 操作数说明 |
|--------|----------|----------|------------|
| 1 | 立即数赋值 | 立即数 → 寄存器 | 参数1：立即数<br>参数2：0<br>目标：寄存器编号 |
| 2 | 寄存器复制 | 寄存器1 → 寄存器2 | 参数1：源寄存器<br>参数2：0<br>目标：目标寄存器 |
| 3 | 加法 | 寄存器1 + 寄存器2 → 目标 | 参数1：源寄存器1<br>参数2：源寄存器2<br>目标：目标寄存器 |
| 4 | 减法 | 寄存器1 - 寄存器2 → 目标 | 参数1：源寄存器1<br>参数2：源寄存器2<br>目标：目标寄存器 |
| 5 | 乘法 | 寄存器1 × 寄存器2 → 目标 | 参数1：源寄存器1<br>参数2：源寄存器2<br>目标：目标寄存器 |
| 6 | 除法 | 寄存器1 ÷ 寄存器2 → 目标 | 参数1：源寄存器1<br>参数2：源寄存器2<br>目标：目标寄存器 |
| 7 | 取模 | 寄存器1 % 寄存器2 → 目标 | 参数1：源寄存器1<br>参数2：源寄存器2<br>目标：目标寄存器 |
| 8 | 比较（等于） | 比较寄存器1和寄存器2是否相等 | 参数1：源寄存器1<br>参数2：源寄存器2<br>目标：标志寄存器 |
| 9 | 比较（不等于） | 比较寄存器1和寄存器2是否不相等 | 参数1：源寄存器1<br>参数2：源寄存器2<br>目标：标志寄存器 |
| 10 | 比较（大于） | 比较寄存器1是否大于寄存器2 | 参数1：源寄存器1<br>参数2：源寄存器2<br>目标：标志寄存器 |
| 11 | 比较（小于） | 比较寄存器1是否小于寄存器2 | 参数1：源寄存器1<br>参数2：源寄存器2<br>目标：标志寄存器 |
| 12 | 比较（大于等于） | 比较寄存器1是否大于等于寄存器2 | 参数1：源寄存器1<br>参数2：源寄存器2<br>目标：标志寄存器 |
| 13 | 比较（小于等于） | 比较寄存器1是否小于等于寄存器2 | 参数1：源寄存器1<br>参数2：源寄存器2<br>目标：标志寄存器 |
| 14 | 逻辑与 | 寄存器1 & 寄存器2 → 目标 | 参数1：源寄存器1<br>参数2：源寄存器2<br>目标：目标寄存器 |
| 15 | 逻辑或 | 寄存器1 \| 寄存器2 → 目标 | 参数1：源寄存器1<br>参数2：源寄存器2<br>目标：目标寄存器 |
| 16 | 逻辑非 | ~寄存器1 → 目标 | 参数1：源寄存器<br>参数2：0<br>目标：目标寄存器 |
| 19 | 写内存 / 硬件 | 将寄存器数据存储到内存地址或硬件 | 参数1：源寄存器<br>参数2：内存地址/硬件地址<br>目标：0 |
| 20 | 读内存 / 硬件 | 从内存地址或硬件读取数据到寄存器 | 参数1：内存地址/硬件地址<br>参数2：0<br>目标：目标寄存器 |
| 21 | 无条件跳转 | 跳转到指定地址 | 参数1：跳转地址<br>参数2：0<br>目标：0 |
| 22 | 为 0 跳转 | 若寄存器值为0则跳转到指定地址 | 参数1：跳转地址<br>参数2：源寄存器<br>目标：0 |
| 23 | 不为 0 跳转 | 若寄存器值不为0则跳转到指定地址 | 参数1：跳转地址<br>参数2：源寄存器<br>目标：0 |
| 24 | 调用函数 / 系统 API | 调用指定地址的函数或系统API | 参数1：函数地址<br>参数2：0<br>目标：0 |
| 25 | 函数返回 | 从函数返回 | 参数1：返回值寄存器<br>参数2：0<br>目标：0 |
| 26 | 退出程序 | 结束程序执行 | 参数1：0<br>参数2：0<br>目标：0 |

### 2.3 寄存器

系统提供16个通用寄存器，编号为0-15。

### 2.4 运行环境

代码由零点编译器编译为各平台原生机器码，无VM、无解释器、无运行时，CPU直接执行。

## 3. 编译器设计

### 3.1 功能需求

1. 词法分析：解析输入的数字指令
2. 语法分析：验证指令格式是否正确
3. 语义分析：检查指令参数是否合法
4. 代码生成：生成各平台原生机器码
5. 打包：生成对应平台的可执行文件
6. 错误处理：提供清晰的错误信息

### 3.2 支持平台

- Windows (x86_64) → .exe
- Linux (x86_64/ARM64) → ELF
- Android (ARM64) → 可执行程序 / 动态库
- 单片机 (ARM Cortex-M) → .bin/.hex

### 3.3 实现方案

编译器将采用以下步骤处理代码：
1. 读取输入文件或命令行输入
2. 按行解析指令
3. 验证每条指令的格式和参数
4. 生成对应架构的机器码
5. 打包成对应平台的可执行文件

### 3.4 技术栈

- 开发语言：Go（跨平台，支持交叉编译）
- 构建工具：Go modules
- 依赖库：标准库为主，可能需要少量第三方库

## 4. 代码生成器设计

### 4.1 功能需求

1. 根据用户需求生成符合零点语言规则的代码
2. 支持基本的算术运算
3. 支持条件判断和跳转
4. 支持输入输出操作
5. 支持循环结构

### 4.2 实现方案

代码生成器将提供以下功能：
1. 命令行界面，接收用户输入的需求
2. 根据需求生成对应的零点语言代码
3. 验证生成的代码是否符合语言规则
4. 输出生成的代码

### 5.4 示例4：乘法运算

代码：
```

10 3 0 0
15 0 0 0
```

### 5.5 示例5：除法运算

需求：计算10÷2并输出结果

代码：
```
1 10 0 1
1 2 0 2
5 1 2 3
10 3 0 0
15 0 0 0
```

### 5.6 示例6：条件判断

需求：比较两个数的大小，输出较大的数

代码：
```
1 10 0 1  // 加载10到r1
1 20 0 2  // 加载20到r2
6 1 2 3   // 比较r1和r2
9 10 0 0  // 如果r1 > r2，跳转到地址10
1 2 0 4   // 否则，r4 = r2
10 4 0 0  // 输出r4
15 0 0 0  // 结束
1 1 0 4   // 地址10：r4 = r1
10 4 0 0  // 输出r4
15 0 0 0  // 结束
```

### 5.7 示例7：循环

需求：输出1到5的数字

代码：
```
1 1 0 1   // 计数器r1 = 1
1 5 0 2   // 最大值r2 = 5
10 1 0 0  // 输出r1
2 1 1 1   // r1 = r1 + 1
6 1 2 3   // 比较r1和r2
8 3 0 0   // 跳转到地址3
15 0 0 0  // 结束
```

### 5.8 示例8：输入输出

需求：输入一个数，输出其平方

代码：
```
11 0 0 0  // 输入到r0
4 0 0 1   // r1 = r0 × r0
10 1 0 0  // 输出r1
15 0 0 0  // 结束
```

## 8. 使用指南

### 8.1 编译器安装

1. **安装Go语言环境**：确保安装了Go 1.20或更高版本
2. **构建编译器**：
   ```bash
   cd compiler
   go build -o zero-compiler.exe main.go compiler.go
   ```
3. **添加到PATH**：将编译器可执行文件所在目录添加到系统PATH环境变量

### 8.2 编译零点语言代码

#### 8.2.1 基本用法

```bash
# 编译为Windows x86_64可执行文件
zero-compiler --platform windows-x86_64 <input_file> -o <output.exe>

# 编译为Linux x86_64可执行文件
zero-compiler --platform linux-x86_64 <input_file> -o <output>

# 编译为ARM64可执行文件
zero-compiler --platform arm64 <input_file> -o <output>

# 编译为Cortex-M固件
zero-compiler --platform cortex-m <input_file> -o <output.bin>
```

#### 8.2.2 命令行选项

| 选项 | 描述 |
|------|------|
| -h, --help | 显示帮助信息 |
| -v, --version | 显示版本信息 |
| --platform <platform> | 指定目标平台 |
| -o <output_file> | 指定输出文件 |
| --debug | 启用调试模式 |
| --optimize | 启用优化 |

### 8.3 代码规范

1. **指令格式**：每条指令必须由4个数字组成，用空格分隔
2. **注释**：以 `//` 开头的行被视为注释
3. **空行**：空行会被忽略
4. **指令码**：指令码必须在0-15范围内
5. **寄存器**：寄存器编号必须在0-15范围内

### 8.4 常见问题

#### 8.4.1 编译错误

- **指令格式错误**：确保每条指令由4个数字组成
- **指令码无效**：指令码必须在0-15范围内
- **寄存器编号无效**：寄存器编号必须在0-15范围内
- **参数错误**：确保各指令的参数符合要求

#### 8.4.2 运行错误

- **段错误**：检查是否访问了无效的内存地址
- **除以零**：确保除法操作的除数不为零
- **无限循环**：检查循环条件是否正确
